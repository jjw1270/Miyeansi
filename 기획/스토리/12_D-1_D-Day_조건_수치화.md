# D-1/D-Day 조건 수치화

이 문서는 D-1 밤 정산과 D-Day 낮 보정, 폐장 후 최종 분기를 실제 게임 시스템으로 옮기기 위한 조건 기준 v0.1이다.

목표는 `충분하다`, `부족하다`로 적혀 있던 감정 조건을 구현 가능한 점수·불리언·우선순위로 정리하는 것이다. 수치는 최종 밸런스가 아니라 스토리 구조를 흔들지 않기 위한 1차 기준이다.

## 1. 기본 원칙

- 수치는 플레이어에게 직접 노출하지 않는다.
- 게임 내에서는 재윤의 독백, 하연의 반응, 기존 인물의 대사, 선택지 잠김/변형으로만 상태를 보여준다.
- D-1 밤은 `현재 상태 진단`이다.
- D-Day 낮은 `마지막 보정`이다.
- D-Day 폐장 후에는 더 이상 보정하지 않고 분기만 판정한다.
- 진엔딩은 하연 점수만 높다고 열리지 않는다. 하연 신뢰, 하연 동선 이해, 기존 관계 정리, 사고 단서, 회피 누적이 모두 기준을 통과해야 한다.

## 2. 상태 변수 기준

### 2.1 하연 축

| 변수 | 범위 | 의미 | 주요 상승 시점 |
| --- | --- | --- | --- |
| `HaYeonTrust` | 0~8 | 하연이 재윤에게 자기 침묵과 속도를 맡길 수 있는 신뢰 | D-25, D-24, D-18~D-16, D-13, D-7, D-2, D-1, D-Day 낮 하연 선택 |
| `HaYeonPaceRead` | 0~4 | 재윤이 하연의 조용한 자리 선호와 동선을 읽는 정도 | D-17, D-16, D-2, D-1, D-Day 낮 하연 선택 |
| `D2PremonitionPassed` | true/false | D-2 말하지 못한 예감에서 하연을 성급히 몰아붙이지 않고 이해했는지 | D-2 |
| `AccidentClueHaYeon` | true/false | 사고 기억이 `하연에게 가려던 선택`과 연결되는 핵심 조각 | D-2, D-1, D-Day 낮 하연 선택 |

하연 축 판정:

| 상태 | 조건 | 의미 |
| --- | --- | --- |
| 안정 | `HaYeonTrust >= 6` 그리고 `HaYeonPaceRead >= 3` | 폐장 후 하연이 있을 곳을 알아차릴 수 있음 |
| 보정 가능 | `HaYeonTrust == 5` 또는 `HaYeonPaceRead == 2` | D-Day 낮 하연 선택으로 보정 가능 |
| 위험 | `HaYeonTrust <= 4` 또는 `HaYeonPaceRead <= 1` | D-Day 낮 한 번으로는 회복이 어려움 |
| 실패 확정 | `D2PremonitionPassed == false` | 하연의 침묵과 동선을 읽지 못해 폐장 후 실패 쪽으로 기움 |

## 3. 기존 관계 축

### 3.1 관계 정리 변수

| 변수 | 범위 | 의미 |
| --- | --- | --- |
| `SoHaResolved` | true/false | 과거로 돌아가 숨는 길을 정리했는지 |
| `SeoRinResolved` | true/false | 정답과 순서 뒤에 숨는 길을 정리했는지 |
| `MiRuResolved` | true/false | 필요한 사람 역할 뒤에 숨는 길을 정리했는지 |

### 3.2 압박 변수

| 변수 | 범위 | 의미 | 높을 때 |
| --- | --- | --- | --- |
| `SoHaPressure` | 0~3 | 소하와의 과거 미련 압박 | 예전처럼 돌아가는 선택지가 강해짐 |
| `SeoRinControl` | 0~3 | 서린에게 선택권을 맡기는 정도 | 안전한 정답 뒤에 숨는 선택지가 강해짐 |
| `MiRuDependence` | 0~3 | 미루에게 필요한 사람으로 남으려는 정도 | 누군가를 두고 가는 죄책감이 강해짐 |

기존 관계 판정:

| 상태 | 조건 | 의미 |
| --- | --- | --- |
| 정리 완료 | 세 `Resolved` 모두 true | 진엔딩 조건 B 충족 |
| 1축 보정 가능 | 미해결 1개 + 해당 압박값 `<= 2` | D-Day 낮 해당 인물 선택으로 마지막 보정 가능 |
| 위험 | 미해결 1개 + 해당 압박값 `3` | D-Day 낮 실패 시 해당 새드엔딩 우선 |
| 실패 위험 큼 | 미해결 2개 이상 | D-Day 낮 한 번으로 모두 보정 불가. 진엔딩 선택지 잠김 |

## 4. 사고 단서 축

사고 단서는 단순 정보가 아니라 `다친 기억까지 안고 현실로 돌아갈 수 있는 기억 조각`이다.

| 변수 | 범위 | 의미 |
| --- | --- | --- |
| `AccidentClueSoHa` | true/false | 재윤이 축제날 하연을 향했다는 감정 단서 |
| `AccidentClueSeoRin` | true/false | 소무대 임시 구조물/조명 프레임 위험 단서 |
| `AccidentClueMiRu` | true/false | 출입 제한 표지/고정끈/보고 대상 단서 |
| `AccidentClueHaYeon` | true/false | 서로 말하려던 장소와 끊긴 문장 단서 |
| `AccidentClueScore` | 0~4 | 위 네 단서의 true 개수 |

사고 단서 판정:

| 상태 | 조건 | 의미 |
| --- | --- | --- |
| 안정 | `AccidentClueScore >= 3` 그리고 `AccidentClueHaYeon == true` | 사고를 하연에게 가려던 선택의 결과로 연결 가능 |
| 보정 가능 | `AccidentClueScore == 2` 그리고 `AccidentClueHaYeon == true` | D-Day 낮 조사/인물 선택으로 1개 보강 가능 |
| 감정 연결 부족 | `AccidentClueScore >= 3` 하지만 `AccidentClueHaYeon == false` | 장소와 물건은 알아도 하연에게 가려던 의미가 비어 있음 |
| 정보 부족 | `AccidentClueScore <= 1` | D-Day 낮 한 번으로는 사고 의미 연결이 어려움 |

## 5. 회피 축

| 변수 | 범위 | 의미 | 주요 상승 |
| --- | --- | --- | --- |
| `Avoidance` | 0~5 | 중요한 선택과 대화를 미룬 누적치 | 답장 미루기, 혼자 확인하기, 애매한 대답, 하연/기존 관계 모두 회피 |

회피 판정:

| 상태 | 조건 | 의미 |
| --- | --- | --- |
| 안정 | `Avoidance <= 2` | 폐장 후 자기 선택을 말할 수 있음 |
| 경고 | `Avoidance == 3` | 다른 조건이 충분하면 통과하지만 고백 전 흔들림 대사 추가 |
| 위험 | `Avoidance >= 4` | 하연 조건이 충분해도 중요한 말을 미룰 위험. 실패/루프 쪽 선택지가 강해짐 |

## 6. D-1 밤 정산

D-1 밤은 축제 1일차 `쉬어가 카페` 운영이 끝난 뒤 실행한다.

### 6.1 정산 항목

| 정산 항목 | 안정 기준 | D-Day 낮 보정 가능 기준 | 위험 기준 |
| --- | --- | --- | --- |
| 하연 신뢰 | `HaYeonTrust >= 6` | `HaYeonTrust == 5` | `HaYeonTrust <= 4` |
| 하연 동선 이해 | `HaYeonPaceRead >= 3` | `HaYeonPaceRead == 2` | `HaYeonPaceRead <= 1` |
| D-2 예감 | `D2PremonitionPassed == true` | 없음 | false면 실패 위험 큼 |
| 기존 관계 정리 | 세 `Resolved` true | 미해결 1개 + 압박 `<= 2` | 미해결 2개 이상 또는 압박 3 |
| 사고 단서 | `AccidentClueScore >= 3` + `AccidentClueHaYeon` | `Score == 2` + `AccidentClueHaYeon` | `Score <= 1` 또는 하연 단서 없음 |
| 회피 | `Avoidance <= 2` | `Avoidance == 3` | `Avoidance >= 4` |

### 6.2 D-1 밤 상태 등급

| 등급 | 조건 | D-Day 낮 처리 |
| --- | --- | --- |
| `Ready` | 모든 항목 안정 | 낮 선택은 확인/대사 변주. 폐장 후 성공 가능 |
| `OneAxisShort` | 보정 가능 항목이 정확히 1개 | 낮 선택으로 해당 축을 보정하면 성공 가능 |
| `TwoAxisShort` | 보정 가능 항목 2개 또는 위험 항목 1개 | 낮 선택 하나로 전부 보정 불가. 진엔딩 선택지 잠김 가능 |
| `CriticalShort` | D-2 실패, 하연 위험, 미해결 2개 이상, 사고 정보 부족, 회피 위험 중 하나 | 폐장 후 실패/새드/루프 우선 |

### 6.3 D-1 밤 연출 규칙

수치를 UI로 보여주지 않고 아래처럼 표현한다.

| 부족 축 | 연출 |
| --- | --- |
| 하연 신뢰 부족 | 하연의 말이 짧아지고, 재윤 독백이 `묻지 못했다`로 끝남 |
| 동선 이해 부족 | 하연이 바라본 방향을 재윤이 놓침 |
| 기존 관계 미정리 | 해당 인물의 목소리나 메시지가 밤 정산에 끼어듦 |
| 사고 단서 부족 | 소무대 이미지가 장소만 남고 이유가 비어 있음 |
| 회피 높음 | 선택지 텍스트가 `내일 생각한다`, `괜찮은 척한다`처럼 미루는 말투로 변함 |

## 7. D-Day 낮 보정

D-Day 낮 선택은 한 번만 가능하다. 선택한 축만 보정하고, 다른 축은 그대로 확정한다.

### 7.1 낮 선택별 보정량

| 낮 선택 | 보정 조건 | 성공 보정 | 한계 |
| --- | --- | --- | --- |
| 하연과 축제를 둘러본다 | `HaYeonTrust >= 5` 또는 `HaYeonPaceRead >= 2` | `HaYeonTrust +1`, `HaYeonPaceRead +1`, 조건 충족 시 `AccidentClueHaYeon = true` | 기존 관계 미해결은 해결하지 못함 |
| 소하와 마지막으로 오래된 길을 걷는다 | `SoHaPressure <= 2` | 올바른 선택 시 `SoHaResolved = true`, `AccidentClueSoHa = true` | 하연 신뢰/동선은 보정하지 못함 |
| 서린의 방송 마무리를 돕는다 | `SeoRinControl <= 2` | 올바른 선택 시 `SeoRinResolved = true`, `AccidentClueSeoRin = true` | 자기 선택을 서린에게 맡기면 실패 |
| 미루의 도움 인원 마무리를 확인한다 | `MiRuDependence <= 2` | 올바른 선택 시 `MiRuResolved = true`, `AccidentClueMiRu = true` | 대신 책임지면 실패 |
| 혼자 야외 소무대 쪽을 본다 | `AccidentClueScore == 2` 이상 권장 | 부족한 물리 단서 1개 보강 가능 | `AccidentClueHaYeon`은 단독 조사로 획득 불가. 회피가 높으면 `Avoidance +1` |

### 7.2 낮 선택 실패 처리

| 실패 상황 | 결과 |
| --- | --- |
| 하연 선택에서 축제 놀이에만 집중 | 하연 신뢰/동선 보정 없음 |
| 소하 선택에서 애매하게 말함 | `SoHaResolved` false 유지, `SoHaPressure` 3으로 상승 가능 |
| 서린 선택에서 서린 안을 그대로 따름 | `SeoRinResolved` false 유지, `SeoRinControl` 3으로 상승 가능 |
| 미루 선택에서 대신 책임짐 | `MiRuResolved` false 유지, `MiRuDependence` 3으로 상승 가능 |
| 혼자 선택으로 단서만 확인하고 공유하지 않음 | 물리 단서는 늘 수 있으나 하연 연결 부족. `Avoidance +1` 가능 |

## 8. 폐장 후 최종 판정

D-Day 폐장 후에는 아래 순서로 판정한다.

### 8.1 진엔딩 필수 조건

```text
HaYeonTrust >= 6
HaYeonPaceRead >= 3
D2PremonitionPassed == true
SoHaResolved == true
SeoRinResolved == true
MiRuResolved == true
AccidentClueScore >= 3
AccidentClueHaYeon == true
Avoidance <= 3
히든 붕괴 조건 미충족
```

이 조건을 모두 만족하면 `[하연에게 말하러 간다]` 또는 이후 실제 스크립트에서 사용할 진엔딩 선택지가 열린다.

### 8.2 분기 우선순위

| 우선순위 | 조건 | 분기 |
| --- | --- | --- |
| 1 | 히든 붕괴 조건 충족 | 히든 세계 붕괴 엔딩 |
| 2 | `D2PremonitionPassed == false` 또는 `HaYeonTrust < 6` 또는 `HaYeonPaceRead < 3` | 하연 실패 루프 |
| 3 | `SoHaResolved == false` 그리고 `SoHaPressure == 3` | 소하 새드엔딩 |
| 4 | `SeoRinResolved == false` 그리고 `SeoRinControl == 3` | 서린 새드엔딩 |
| 5 | `MiRuResolved == false` 그리고 `MiRuDependence == 3` | 미루 새드엔딩 |
| 6 | 미해결 관계가 1개 이상 남음 | 기존 관계 미정리 루프 또는 해당 인물 새드엔딩 약화판 |
| 7 | `AccidentClueScore < 3` 또는 `AccidentClueHaYeon == false` | 사고 의미 미연결 루프 |
| 8 | `Avoidance >= 4` | 회피 루프. 하연에게 가려 하지만 마지막 말을 미룸 |
| 9 | 모든 필수 조건 충족 | 진엔딩 |

우선순위 6의 약화판은 실제 구현 단계에서 새드엔딩 컷을 짧게 처리할지, D-25 루프 복귀로 처리할지 결정한다. 현재 스토리 기준에서는 강한 압박값 3이 아니면 완전 새드엔딩보다 루프 복귀에 가깝다.

## 9. 선택지 노출 기준

### 9.1 폐장 후 선택지 세트

| 상태 | 선택지 노출 |
| --- | --- |
| 모든 조건 충족 | `[하연에게 말하러 간다]` 노출 |
| 하연 조건 부족 | `[소무대 쪽으로 가본다]`만 노출. 이유를 모름 |
| 기존 관계 압박 높음 | 해당 도피 선택지가 함께 노출. 예: `[예전처럼 돌아간다]` |
| 사고 단서 부족 | `[확인하러 간다]`처럼 조사형 선택지만 노출. 마음 고백 선택지 없음 |
| 회피 높음 | `[내일 말한다]`, `[아직은 말하지 않는다]` 같은 미루는 선택지 노출 |

### 9.2 진엔딩 선택지 명칭 후보

실제 스크립트 분할 전 기준 후보:

```text
[하연에게 말하러 간다]
```

조금 더 결말 선택지에 가깝게 쓰는 후보:

```text
[이번에는 내가 먼저 간다]
```

최종 진엔딩 선택지 후보:

```text
[다친 기억까지 안고 하연에게 돌아간다]
```

D-Day 실제 스크립트 분할 문서에서는 위 셋 중 장면 톤에 맞는 하나를 고른다.

## 10. 예시 케이스

| 케이스 | 상태 | 낮 선택 | 최종 분기 |
| --- | --- | --- | --- |
| 안정 진엔딩 | 하연 6/3, 세 관계 정리, 단서 3+하연, 회피 2 | 아무 선택이나 확인 장면 | 진엔딩 가능 |
| 하연만 1 부족 | 하연 5/3, 세 관계 정리, 단서 충분 | 하연 선택 성공 | 진엔딩 가능 |
| 관계 1개 미정리 | 하연 충분, 소하 미정리+압박2, 단서 충분 | 소하 선택 성공 | 진엔딩 가능 |
| 단서 1개 부족 | 하연 충분, 관계 정리, 단서2+하연 | 서린/미루/혼자 중 부족 단서 보정 | 진엔딩 가능 |
| D-2 실패 | D2PremonitionPassed false | 하연 선택을 해도 회복 불가 | 하연 실패 루프 |
| 하연 단서 없음 | 단서3이지만 AccidentClueHaYeon false | 혼자 조사로 회복 불가 | 사고 의미 미연결 루프 또는 하연 실패 루프 |
| 미정리 2개 | 소하/미루 미정리 | 낮 한 번으로 모두 해결 불가 | 진엔딩 잠김, 압박 높은 쪽 우선 |
| 회피 높음 | Avoidance 4 | 어떤 선택을 해도 말하기 직전 미룸 | 회피 루프 |

## 11. 실제 구현 메모

- `HaYeonTrust`와 `HaYeonPaceRead`는 정수형으로 관리한다.
- `Resolved`와 `AccidentClue*`는 우선 불리언으로 관리한다.
- `AccidentClueScore`는 저장 변수보다 평가 시 계산값으로 두는 편이 안전하다.
- D-1 밤에는 `DDayReadinessGrade` 같은 임시 평가 결과를 생성할 수 있다.
- D-Day 낮 선택 후에는 동일 평가 함수를 한 번 더 돌려 최종 분기만 결정한다.
- 실제 UI에는 점수와 등급을 노출하지 않고, 선택지 잠김/문장 변화/회상 컷으로만 표현한다.
