# 개발 일정

이 문서는 **「너에게 닿는 마지막 여름」**의 개발 진행 순서와 현재 상태를 확인하기 위한 작업 보드다.

세부 설계는 각 정본 문서를 따르고, 이 문서는 `무엇을 먼저 구현할지`, `어디까지 완료로 볼지`, `관련 기획서를 어디서 찾을지`를 정리한다.

## 현재 단계

| 항목 | 상태 |
| --- | --- |
| 현재 단계 | `VisualNovelPlugin` EventHub 1차 구조 구현 완료 |

## 공유 방식

공유용 진행 상태는 이 Markdown 문서에 직접 기록한다.

- `개발_일정.md`의 상태값이 정본이다.
- Markdown을 수정한 뒤 통합 웹 기획서를 재생성하면 `index.html`과 `기획/통합_기획서.html`에 같은 상태가 반영된다.
- 변경 내용을 GitHub Pages 브랜치에 푸시하면 다른 사람도 같은 개발 일정을 볼 수 있다.
- 별도 서버 없이 공개하려면 이 방식을 사용한다.
- 여러 사람이 동시에 상태를 수정해야 하는 단계가 오면 GitHub Issues/Projects 또는 별도 백엔드 저장소를 검토한다.

## 로컬 편집 방식

로컬 웹에서는 상태 칸이 드롭다운으로 표시된다.

> 주의: 드롭다운 변경은 내 브라우저에만 저장되며, 공유 페이지에는 반영되지 않는다.

- 드롭다운에서 고른 상태는 현재 브라우저의 로컬 저장소에 저장된다.
- 새로고침해도 로컬에서 고른 상태는 유지된다.
- 이 값은 GitHub Pages 방문자에게 공유되지 않는다.
- 공개용 상태로 확정할 때만 `개발_일정.md`의 상태값을 같은 값으로 맞춘 뒤 웹 기획서를 재생성한다.

## 업데이트 규칙

- 실제 구현이 완료되면 `상태`를 `완료`로 바꾼다.
- 진행 중인 항목은 `진행중`, 확인 단계는 `검증중`, 미루는 항목은 `보류`로 표시한다.
- 완료 기준이 바뀌면 이 문서보다 기준 문서를 먼저 수정한다.
- 구현 중 세부 설계가 바뀌면 [VisualNovelPlugin 데이터 상세 설계](./시스템/VisualNovelPlugin_데이터_상세설계.md)를 우선 갱신한다.
- 통합 웹 기획서는 Markdown 수정 후 `python 기획/build_planning_web.py`로 재생성한다.
