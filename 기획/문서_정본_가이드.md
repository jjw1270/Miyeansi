# 기획 문서 정본 가이드

이 문서는 `기획/` 아래 문서들의 역할과 정본 기준을 정리한다.

목표는 모든 내용을 하나의 거대한 Markdown에 합치는 것이 아니라, 원본 문서는 주제별로 분리하고 통합 웹은 읽기용 생성물로 유지하는 것이다.

## 1. 문서 운용 원칙

- Markdown 원본이 정본이다.
- `통합_기획서.html`은 원본 Markdown을 모아 만든 읽기용 생성물이다.
- 루트 `index.html`은 GitHub Pages 진입용 생성물이며, `통합_기획서.html`과 같은 내용을 가진다.
- 큰 방향은 README와 요약 문서에 두고, 세부 조건·대본·수치 기준은 전용 문서에 둔다.
- 같은 내용을 여러 문서에 길게 복제하지 않는다. 중복이 필요한 경우 한 문서만 정본으로 두고 다른 문서는 링크와 요약만 둔다.
- 오래된 문서가 최신 정본과 충돌하거나 중복되면 역할 표시로 남기지 말고, 링크/참조를 확인한 뒤 삭제하거나 최신 정본으로 흡수한다.

## 6. 오래된 문서 삭제 기준

현재 삭제 대상 문서는 없다.

아래 기준에 해당하면 역할 표시로 보존하지 않고, 최신 정본에 필요한 내용을 흡수한 뒤 삭제한다.

| 상태 | 처리 |
| --- | --- |
| 최신 문서와 내용이 거의 같은 오래된 문서 | 최신 정본에 없는 내용만 옮긴 뒤 오래된 문서 삭제 |
| 상위 요약으로 유효한 문서 | `요약 인덱스`로 표시하고 상세 정본 링크 보강 |
| 생성물 | `직접 수정 금지` 표시 또는 README에서 생성물로 분류 |
| 실제로 불필요한 문서 | 링크/참조 여부 확인 후 삭제 |

`스토리_컨셉.md`는 삭제 대상이 아니라 상위 요약 인덱스로 유지한다. 세부 정본은 `스토리/00~17` 문서다.

## 7. 통합 웹 갱신 규칙

Markdown 원본을 수정한 뒤 아래 명령으로 통합 웹을 재생성한다.

```powershell
python 기획/build_planning_web.py
```

검증 기준:

- 생성 로그의 Markdown 문서 수 확인
- 새 문서가 통합 웹 데이터에 포함되는지 확인
- Markdown 내부 링크 누락 여부 확인
- 루트 `index.html`과 `통합_기획서.html`은 직접 수정하지 않고 생성 결과만 커밋한다.
